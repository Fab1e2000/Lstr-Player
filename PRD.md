# Lstr Player MVP PRD

版本：1.0  
状态：MVP 构建依据  
目标平台：macOS 26+  
最后更新：2026-06-02

## 1. 产品概述

Lstr Player 是一款运行在 macOS 26+ 上的本地音频播放器，用于播放用户本地工作区中的音乐或同人音声专辑。

MVP 的核心目标是：读取一个本地工作区，将工作区内的一级子文件夹识别为专辑，并提供极简的专辑选择、专辑播放、菜单栏歌词显示和基础播放控制能力。

## 2. MVP 范围

本 PRD 中已经列出的需求全部属于 MVP 范围。

MVP 不包含以下能力：

- 搜索功能。
- 多工作区管理。
- 读取多层嵌套文件夹。
- 自定义排序。
- 退出后记忆播放位置。
- 播放页面歌词显示。
- 自定义应用图标。

## 3. 技术栈

MVP 使用以下技术栈：

- Swift。
- SwiftUI。
- AVFoundation。
- FileManager。
- UserDefaults。
- 少量 AppKit interop。

技术栈约束：

- UI 使用 SwiftUI 构建。
- 音频播放、音频元数据读取使用 AVFoundation。
- 本地文件夹读取使用 FileManager。
- 唯一工作区配置使用 UserDefaults 保存。
- macOS 菜单栏歌词显示、窗口生命周期等 SwiftUI 不便直接处理的部分允许使用少量 AppKit interop。
- MVP 暂不引入数据库。

## 4. 用户界面

### 4.1 整体风格

UI 必须使用 SwiftUI。

界面需要积极适配 macOS 26 的液态玻璃特性。

图标、按钮和界面元素需要保持风格统一。

图标、按钮和界面元素的尺寸、间距需要符合现代 macOS 应用的美学。

按钮风格使用 SwiftUI 的现代风格。

### 4.2 应用图标

应用图标待定。

MVP 阶段使用默认构建图标。

### 4.3 页面层级

用户点击软件图标进入软件后，进入专辑选择页面。

用户点击专辑选择页面中的专辑元素后，进入对应专辑的播放页面。

用户点击播放页面中的歌曲播放条目后，歌曲立即开始播放。

页面层级为：

1. 专辑选择页面。
2. 专辑播放页面。

### 4.4 启动与空状态

首次打开软件或没有可显示内容时，软件显示空状态。

空状态不显示专辑元素。

### 4.5 专辑选择页面

专辑选择页面用于展示当前工作区内识别到的专辑。

专辑选择页面中的每个专辑元素必须是正方形矩形。

专辑元素的图像来自专辑文件夹内的封面图片。

专辑封面需要规范化后展示：

- 如果原图是正方形，则不做裁切。
- 如果原图不是正方形，则从原图内部裁切最大居中正方形作为最终展示图像。

专辑选择页面上方需要提供刷新按钮。

刷新按钮用于检测新增专辑。

点击刷新按钮后，软件静默刷新，并立即更新专辑选择页面。

### 4.6 专辑播放页面

专辑播放页面必须包含：

- 专辑封面。
- 歌曲播放条目列表。
- 播放进度条。
- 播放 / 暂停按钮。
- 上一首按钮。
- 下一首按钮。
- 播放模式按钮。

播放进度条周边应放置播放 / 暂停、上一首、下一首、播放模式按钮。

播放模式按钮必须支持：

- 单曲循环。
- 列表播放。

专辑播放页面不显示歌词。

### 4.7 页面切换

专辑选择页面和专辑播放页面之间的切换需要流畅平稳。

专辑选择页面和专辑播放页面之间的切换需要具有流动感。

## 5. 工作区与文件夹读取

### 5.1 Settings 入口

软件在 macOS 菜单栏中需要提供 Settings 入口。

点击软件名称后的下拉菜单中，需要存在 Settings 选项。

Settings 中可以修改软件读取的工作区文件夹。

### 5.2 工作区规则

软件只允许设置一个工作区。

修改工作区文件夹后，软件必须立即重新扫描。

软件需要申请权限读取工作区文件夹。

### 5.3 专辑识别规则

工作区内的所有一级子文件夹都被认为是一张专辑。

工作区文件夹本身不被认为是一张专辑。

软件不读取更深层文件夹。

专辑选择页面中的专辑使用默认排序。

如果没有可读取内容，则不显示。

### 5.4 专辑内容

每张专辑文件夹中可能存在：

- 音频文件。
- 封面图片。
- LRC 歌词文件。
- 其他杂项文件。

软件需要忽略不属于 MVP 处理范围的其他杂项文件。

### 5.5 支持的音频格式

MVP 支持以下音频格式：

- FLAC。
- MP3。
- WAV。

## 6. 封面识别

### 6.1 封面来源

软件需要从专辑文件夹中提取封面文件。

如果专辑文件夹中存在多份封面文件，则读取文件名排序后的第一个封面文件。

支持的封面图片格式：

- JPG。
- PNG。

### 6.2 缺失封面

如果专辑缺失封面文件，则显示规定的缺失图标。

### 6.3 内嵌封面

软件需要能够提取歌曲内部封装的内嵌封面信息。

## 7. 歌曲识别与排序

### 7.1 歌曲识别

软件需要正确识别专辑文件夹中的音频文件。

识别范围仅限 MVP 支持的音频格式。

### 7.2 歌曲显示名称

软件需要删除歌曲文件名前面的数字标识，并将处理后的名称作为界面显示名称。

示例：

- 原文件名：`1. afterworld`
- 界面显示名：`afterworld`

该处理不能更改原始文件名称。

### 7.3 歌曲排序

歌曲优先按照音频内部封装的音轨号排序。

软件需要能够提取歌曲内部封装的音轨号。

## 8. 播放功能

### 8.1 点击播放

用户点击歌曲播放条目后，该歌曲立即播放。

点击歌曲播放条目后，软件继续播放该歌曲后续歌曲。

### 8.2 播放控制

专辑播放页面需要支持：

- 播放。
- 暂停。
- 上一首。
- 下一首。
- 拖动进度条跳转。
- 单曲循环。
- 列表播放。

### 8.3 播放进度

播放进度条允许拖动跳转。

切歌后，播放进度归零。

退出软件后，不记忆播放位置。

### 8.4 不可播放文件

如果音频不可播放，软件需要报错并取消播放。

## 9. 歌词功能

### 9.1 LRC 匹配

LRC 歌词文件按照同名匹配。

### 9.2 菜单栏歌词

播放歌曲时，软件需要在 macOS 菜单栏实时显示歌词。

如果当前歌曲有歌词，则实时显示歌词。

如果当前歌曲没有歌词，则显示“无歌词”。

### 9.3 播放页面歌词

专辑播放页面不显示歌词。

## 10. 窗口与应用生命周期

关闭窗口不退出软件。

关闭窗口后，在软件退出前，播放状态不变。

关闭窗口后，在软件退出前，菜单栏歌词显示不变。

软件使用 `Command + Q` 退出。

## 11. 验收标准

MVP 验收时，至少需要满足以下标准：

- 可以通过 Settings 设置唯一工作区。
- 修改工作区后，软件立即重新扫描。
- 工作区内一级子文件夹被显示为专辑。
- 更深层文件夹不会被读取为专辑。
- 专辑以正方形元素显示在专辑选择页面。
- 专辑封面可以从 JPG 或 PNG 中提取。
- 多个封面存在时，读取文件名排序后的第一个。
- 缺失封面时显示规定的缺失图标。
- 可以识别 FLAC、MP3、WAV 音频文件。
- 歌曲显示名会去掉文件名前面的数字标识。
- 去数字标识不会修改原始文件名称。
- 歌曲优先按照内部音轨号排序。
- 点击专辑后进入专辑播放页面。
- 点击歌曲播放条目后立即播放。
- 当前歌曲播放完成后继续播放后续歌曲。
- 播放 / 暂停、上一首、下一首可用。
- 单曲循环和列表播放可切换。
- 播放进度条可拖动跳转。
- 切歌后播放进度归零。
- 退出软件后不记忆播放位置。
- LRC 歌词按同名匹配。
- 播放时 macOS 菜单栏实时显示歌词或“无歌词”。
- 专辑播放页面不显示歌词。
- 刷新按钮静默刷新并立即更新专辑选择页面。
- 无可读取内容时不显示专辑。
- 音频不可播放时，软件报错并取消播放。
- 关闭窗口后软件不退出，播放和菜单栏歌词保持不变。
- `Command + Q` 可以退出软件。
